Temp

####碎片笔记

  • 浏览器一般包括渲染器和js脚本引擎

    Mozilla Firefox —Gecko
    Safari、Chrome —WebKit
    Opera —Presto
    Intenet Explorer —Trident

  • Js数据类型包括基本数据类型(原始类型)和引用数据类型(对象类型)；

  • 原始类型有：
    1.number
    整型；浮点型(注意精度问题)
    2.string
    放在引号里的大串字符序列：

    * 单双引号可以交替使用
* 总是记混的几个方法：slice(截取，类似于substring)、split[分割，返回数组]

    3.boolean
    任意的JS值都可以转换为布尔值

    * undefined,null,0,-0,NaN,""(空字符串)--->false
* 除以上外的所有值--->true

    4.null是object类型
    5.undefined未定义

  • 对象类型：Array,Function,Date,RegExp,Error

    对象是属性的集合，每个属性是有‘名/值对’构成；

    普通的Js对象是‘命名值’的无序集合;

    var arr = [1, 2, 3];
arr.length; // 3
arr.length = 6;
arr; // arr变为[1, 2, 3, undefined, undefined, undefined]
arr.length = 2;
arr; // arr变为[1, 2]
var arr = [1, 2, 3];
arr[5] = 'x';
arr; // arr变为[1, 2, 3, undefined, undefined, 'x']
大多数其他编程语言不允许直接改变数组的大小，越界访问索引会报错。然而，JavaScript的Array却不会有任何错误。在编写代码时，不建议直接修改Array的大小，访问索引时要确保索引不会越界.
实际上JavaScript对象的所有属性都是字符串，不过属性对应的值可以是任意数据类型。
JavaScript规定，访问不存在的属性不报错，而是返回undefined.
delete object.key;//删除对象属性
'key' in object;//检测是否拥有某一属性
要判断一个属性是否是object自身拥有的，而不是继承得到的，可以用hasOwnProperty()方法;

  • Tips:

    • 在内存中存储方式不一样：前者存储在栈内存(Stack)中，后者存储在堆内存(heap)中

    • 所以，前者值不可更改，后者可以

      类型转换：

      除了null和undefined不能进行类型转换外，其他所有类型的值都可以转换成其他类型。

    • string/number/boolean直接使用时，存在隐式转换；

    • toString()方法返回一个反映这个对象的字符串；
    • valueOf()方法返回对象本身；

Interview

发表于 2016-10-25   |   分类于 心情随笔   |     |   阅读次数

前言

网申不是抱着太大的期望，而且网申时隔也有点久远，对苏小妍的认识也越来越清晰。认识的人中想去苏小妍的主要原因有国企情节、想在苏州发展，看中待遇和工作环境。其实，除了第一点，后些确实是苏小妍的强有力优势。

面试

我的一面安排在今天的下午12:30，而且我到基本也是下午后一批。按规定两个小组同时进行，每个组十人。我们组只有九人，我排在第八。对于不太善言辞的我，心里飘过一丝庆幸，毕竟可以有喘息的机会。2:45我们去面试厅，酒店环境挺好，硬件设施和条件都挺好(baidu告诉我是five-star/(ㄒoㄒ)/)，两位很nice的女面试官，这时候其实心里的紧张忐忑缓和了10%，坐下后左边interviewer很温柔的告诉我们注意事项，然后开始。阅读的案例其实和第一天的议题有点类似，哎苦于最近总是精神无法集中，在3分钟我看了好几遍文字，才去想解决思路，觉得3分钟很快╮(╯▽╰)╭，这时候稍稍感受到自己是排第八的优势，但劣势是大家已基本解释透案例，难以提出辨识度高的见解。每个人1分钟的陈述里，确实是看出每个人的反应和表达能力，轮到我时我居然把握不住时间，没想到1分钟这么快(感觉说的都是废话，而且并不是特流畅)，不知道怎么收尾时，问了句面试官可不可以让我很快阐释完，面试官却温柔的说接下来还有讨论环节可以再说明自己的观点，哎心里是有点难过的，觉得水平太菜惹…

20分钟讨论环节里，我话说的不算少，但回想看，好像都是在否定别人的观点(实在觉得别人的意见不太符合案例)，也不知道这样是不是被面试官认定为不易融入集体的个别，但我觉得如果是技术官，应该会认同我的看法:-D(呵呵，这里就是未知数X)，总结陈词是4号女生完成的，个人觉得她的口才和组织能力还是蛮强的。

接下来，每个人的半结构化问题回答，好久(;-)才轮到我，我的问题主要是为什么要留在苏州，挑选个项目介绍(感觉陈述的没有抓住重点(⊙o⊙)…)，业余时间会干嘛(我答的居然是打羽毛球..)，好像就没有了，其他人的问题主要是学习能力怎么样，自认为自己的亮点是什么，针对个人简历进行了询问(面试官的语气很温和，真的就像是询问，还会给你补充个词什么的)。其实大家(exc me)表现都还挺好,从面试官的语气真的无法辨别对他们哪个不满意…

所得

  • 其实现在(20:05)人是疲的，主要是心理，感觉情商真的很重要(怎么都不够用(ˇˍˇ) )；
  • 没能进BAT-style com.像是给自己重重的一记耳光，警醒活在理想王国的自己；
  • 对人要谦虚、友善；
  • 人之间的差别也是每天在积累的，优秀的人会越优秀；
