终面结束，知道是和苏小妍说再见的时候/(ㄒoㄒ)/~~，心里很难过，任何的努力在此刻都是苍白的，不能达到该有的技术水平再多的修饰弥补都是徒劳。

面试是在上午接近九点开始，三位面试官，其中一位是主要负责人。可能因为是上午第一个面试，面试官可能也还没有特别准备好，先是让介绍下项目中和前端相关的工作，答得就不是很好，其中提到了Bootstrap，面试官顺此提到Bootstrap是用什么语言写的，我答的CSS是LESS写的，于是顺带问了下了不了解LESS，这里回答得应该还可以，然后问了原生的JS API，我居然被问倒了╮(╯▽╰)╭，父元素中添加一个子元素，这是初学者应该要会的，我竟然答得都是什么add、createElement,难怪让我再想想，额真是框架用多了，不会底层毁自己！怎么没想到准备这些！于是后来的问题CSS中怎么来使用浮动，我也是尴尬的，真的不应该啊。还有个问题是从URL输入’baidu’到浏览器返回其中的流程是什么，没回答上来的是HTTP请求的请求头里的参数信息，我还沉默了一会，真是减分项，不会就说不会好了。到这里已经慌了阵脚，感觉到这是一次很严重的失利，暴露了我基本功极不扎实。另一位面试官问的问题是js里的对象怎么理解，我感觉我回答得也比较皮毛，不是他们想听到的答案。之前他还问到Git常用的命令，对了，开始时他们问到的如果不让你用框架怎么来实现响应式布局，以及如果不想要框架里的某一个样式效果怎么办，我回答的是改源代码，女面试官还问到平时怎么学非框架类的前端技术/CSS,主要看什么书。我答的就是写些单页应用之类的(感觉也不是她想要的答案)。最后是问希望什么样的工作环境，我再问的是前端的主要工作内容，主面试官还做了蛮细致的回答。至此面试结束，整个人超沮丧。

反思这次面试经历，我觉得自己太过飘忽了，学技术时没有足够脚踏实地，被现实狠狠的打了脸。

同样有项目经历，他人掌握了技术同时一通百通，为什么我掌握的却如过沙漏，勉强的才有那么一些，学习方法是关键！其实大浪淘沙，你或许不够优秀，但是也似乎忘了该怎么扬长避短，恰如其分的展示和利用！这次面试如一记响亮的耳光，警醒我认清自己，摆正姿态，最重要的是既然认定了便再无岁月可回头！

得之我幸，失之我命。

Front-end Interview Preparation

发表于 2016-11-06   |   分类于 记忆碎片   |     |   阅读次数

面试接踵而至，[遗憾于平时接触新技术时没有很好的记录归纳] 技术总结迫在眉睫，可谓’功在当下、利在千秋’, 因为感觉面试氛围很大程度上取决于所言是否让interviewer感觉赏心悦目[喜上眉梢-or we can say 志同道合，可塑之才~]。

自我介绍

做过哪些项目，挑一个讲一下

团队开发时，如何分工配合；你的工作

技术栈，前端框架

[React、Vue]在项目中怎么用的，组件之间如何交互；
项目中遇到的问题，怎么解决的
为什么要使用框架[echarts等]

  1. Bootstrap

  2. nodeJs

    nodeJs中唯一的全局对象global(相对于浏览器中的window),global.console;

    process代表当前node.js进程；

    Javascript是事件驱动执行的单线程模型，node.js也不例外。Node.js不断执行响应事件的JavaScript函数，直到没有任何响应事件的函数可以执行时，Node.js就退出了。[process.nextTick()]

    判断js执行环境：

     if (typeof(window) === 'undefined') {
console.log('node.js');
} else {
console.log('browser');
}

    Node利用JavaScript的函数式编程的特性，轻而易举地实现了模块的隔离;

    Koa: next generation web framework for nodejs

  1. 模块化；前端组件化

  2. 前后端交互，Ajax，布局，框架

  3. 数据交互类型；相对路径和绝对路径哪个加载速度更快

  4. ie678兼容性问题有哪些

  5. 前端工程的三个阶段：
    根据项目特征进行框架/库的技术选型
    简单构建优化: 选型一种构建工具，对代码进行压缩、校验，再以页面为单位进行简单的机缘合并
    JS/CSS模块化开发：

    分而治之是软件工程中的重要思想，是复杂系统开发和维护的基石，这点放在前端开发中同样适用。在解决了基本开发效率运行效率问题之后，前端团队开始思考维护效率，模块化是目前前端最流行的分治手段。

JS模块化方案很多，AMD/CommonJS/UMD/ES6 Module等，对应的框架和工具也一大堆；CSS模块化开发基本都是在less、sass、stylus等预处理器的import/mixin特性支持下实现的组件化开发与资源管理

前端相比其他软件开发，在基础架构上更迫切的需要组件化开发和资源管理和选型技术、如何定制规范、如何分治系统、如何优化性能、如何加载资源，当你从切图开始转变为思考这些问题的时候，我想说：你好，工程师！

如何学习新技术

学的最有成就的一门技术是什么；

  1. 数组方法：

    var arr = [1,2,3,4,5];
    arr.push(6);
    arr.pop(6);
    arr.unshift(0);
    arr.shift(0);

    arr.join(); //即arr.join(‘,’)

  1. 立即执行函数使用块作用域申明来防止污染全局变量

  2. requireJs防止js加载阻塞页面渲染，使用程序调用的方式加载js;其目标是鼓励代码的模块化，使用不同于传统标签的脚本加载步骤，用来加速、优化代码，但主要目的是代码的模块化。
    define用来定义一个模块，require加载依赖模块，并执行加载完后的回调函数；
    加载requireJs脚本的script标签加入data-main属性，这个属性指定的js将在加载require.js后处理我们把require.config的配置加入到data-main后，就可以使每一个页面都使用这个配置，然后页面中就可以直接使用require来加载所有的短模块名
    data-main还有一个重要的功能，当script标签指定data-main属性时，require会默认的将data-main指定的js为根路径，是什么意思呢？如上面的data-main=”js/main”设定后，我们在使用require([‘jquery’])后(不配置jquery的paths)，require会自动加载js/jquery.js这个文件，而不是jquery.js，相当于默认配置了：

    require.config({
    baseUrl: 'js'
})

  1. typeOf()的返回值有undefined, object, boolean, number, string, function ~
    Javascript中除了number, string, boolean, null, undefined 之外的就是对象了

  2. HTML5中的新标签：section, article, aside, nav, footer, figure, canvas等等

碎片笔记

  • 浏览器一般包括渲染器和js脚本引擎

    Mozilla Firefox —Gecko
    Safari、Chrome —WebKit
    Opera —Presto
    Intenet Explorer —Trident

  • Js数据类型包括基本数据类型(原始类型)和引用数据类型(对象类型)；

  • 原始类型有：
    1.number
    整型；浮点型(注意精度问题)
    2.string
    放在引号里的大串字符序列：

    * 单双引号可以交替使用
* 总是记混的几个方法：slice(截取，类似于substring)、split[分割，返回数组]

    3.boolean
    任意的JS值都可以转换为布尔值

    * undefined,null,0,-0,NaN,""(空字符串)--->false
* 除以上外的所有值--->true

    4.null是object类型
    5.undefined未定义

  • 对象类型：Array,Function,Date,RegExp,Error

    对象是属性的集合，每个属性是有‘名/值对’构成；

    普通的Js对象是‘命名值’的无序集合;

    var arr = [1, 2, 3];
arr.length; // 3
arr.length = 6;
arr; // arr变为[1, 2, 3, undefined, undefined, undefined]
arr.length = 2;
arr; // arr变为[1, 2]
var arr = [1, 2, 3];
arr[5] = 'x';
arr; // arr变为[1, 2, 3, undefined, undefined, 'x']
大多数其他编程语言不允许直接改变数组的大小，越界访问索引会报错。然而，JavaScript的Array却不会有任何错误。在编写代码时，不建议直接修改Array的大小，访问索引时要确保索引不会越界.
实际上JavaScript对象的所有属性都是字符串，不过属性对应的值可以是任意数据类型。
JavaScript规定，访问不存在的属性不报错，而是返回undefined.
delete object.key;//删除对象属性
'key' in object;//检测是否拥有某一属性
要判断一个属性是否是object自身拥有的，而不是继承得到的，可以用hasOwnProperty()方法;

  • Tips:

    • 在内存中存储方式不一样：前者存储在栈内存(Stack)中，后者存储在堆内存(heap)中

    • 所以，前者值不可更改，后者可以

      类型转换：

      除了null和undefined不能进行类型转换外，其他所有类型的值都可以转换成其他类型。

    • string/number/boolean直接使用时，存在隐式转换；

    • toString()方法返回一个反映这个对象的字符串；
    • valueOf()方法返回对象本身；

前言

网申不是抱着太大的期望，而且网申时隔也有点久远，对苏小妍的认识也越来越清晰。认识的人中想去苏小妍的主要原因有国企情节、想在苏州发展，看中待遇和工作环境。其实，除了第一点，后些确实是苏小妍的强有力优势。

面试

我的一面安排在今天的下午12:30，而且我到基本也是下午后一批。按规定两个小组同时进行，每个组十人。我们组只有九人，我排在第八。对于不太善言辞的我，心里飘过一丝庆幸，毕竟可以有喘息的机会。2:45我们去面试厅，酒店环境挺好，硬件设施和条件都挺好(baidu告诉我是five-star/(ㄒoㄒ)/)，两位很nice的女面试官，这时候其实心里的紧张忐忑缓和了10%，坐下后左边interviewer很温柔的告诉我们注意事项，然后开始。阅读的案例其实和第一天的议题有点类似，哎苦于最近总是精神无法集中，在3分钟我看了好几遍文字，才去想解决思路，觉得3分钟很快╮(╯▽╰)╭，这时候稍稍感受到自己是排第八的优势，但劣势是大家已基本解释透案例，难以提出辨识度高的见解。每个人1分钟的陈述里，确实是看出每个人的反应和表达能力，轮到我时我居然把握不住时间，没想到1分钟这么快(感觉说的都是废话，而且并不是特流畅)，不知道怎么收尾时，问了句面试官可不可以让我很快阐释完，面试官却温柔的说接下来还有讨论环节可以再说明自己的观点，哎心里是有点难过的，觉得水平太菜惹…

20分钟讨论环节里，我话说的不算少，但回想看，好像都是在否定别人的观点(实在觉得别人的意见不太符合案例)，也不知道这样是不是被面试官认定为不易融入集体的个别，但我觉得如果是技术官，应该会认同我的看法:-D(呵呵，这里就是未知数X)，总结陈词是4号女生完成的，个人觉得她的口才和组织能力还是蛮强的。

接下来，每个人的半结构化问题回答，好久(;-)才轮到我，我的问题主要是为什么要留在苏州，挑选个项目介绍(感觉陈述的没有抓住重点(⊙o⊙)…)，业余时间会干嘛(我答的居然是打羽毛球..)，好像就没有了，其他人的问题主要是学习能力怎么样，自认为自己的亮点是什么，针对个人简历进行了询问(面试官的语气很温和，真的就像是询问，还会给你补充个词什么的)。其实大家(exc me)表现都还挺好,从面试官的语气真的无法辨别对他们哪个不满意…

所得

  • 其实现在(20:05)人是疲的，主要是心理，感觉情商真的很重要(怎么都不够用(ˇˍˇ) )；
  • 没能进BAT-style com.像是给自己重重的一记耳光，警醒活在理想王国的自己；
  • 对人要谦虚、友善；
  • 人之间的差别也是每天在积累的，优秀的人会越优秀；
