Summary of 2016

蒽，一开始我也不喜欢任何带象征意义的日子，这几日连续看过了网路上你们的感慨总结，嗯，我也酝酿好(Maybe)了告别的勇气。

喜怒哀乐皆可念
虽然我并未预知
青春是一去不回来的 –By木心

我摘下了耳机，25岁这一年是要好好总结过往的。

照例，先是一段流水(捂脸.jpg)。年初匆忙回校后继续赶着项目，继续着2015年下半年开始后一直的节奏，这是我最初拓展知识圈的一段时光，如今回看，觉得真是步履匆忙和疯狂生长的日子。5-6月份基本移交给了公司现在的开发伙伴，但似乎这两个月也没什么里程碑式的成果，想不起我都做了什么事(无奈~)。前几天有机会看到了平台现在的版本，真心蛮不错的呢(赞)！7月份左右开始参与到吴大师兄和我师父(笑~之前觉得叫着别扭，写出来很有武侠气息哈哈哈)的项目。暑假基本是和他们以及实验室小伙伴混在一起，一起吃饭一起写(To me It’s ‘搬’)代码理需求，我知道这是我研究生阶段最有方向感最不害怕彷徨的时期。9月份左右，形势所逼，师父回了老东家，原有的节奏无征兆的被打破，也差不多是这时候我感受到在学习和技术上对他们所产生的依赖性。吴师兄不仅是在技术还有处事沟通等多方面给我带来影响，让我感觉能遇到这位本没有太多交集的师兄真是我的运气指数有five star哈哈。遗憾的是下半学期开学后我更多忙着写(bie)小论文和找工作，并没有帮到师兄太多的忙。10月份以来就是各种找工作，尝试了很多饱尝酸甜苦辣，其中有让我欣喜的时刻，但总的说结果差强人意。一直到现在，我深感自己辜负了自己的经历，配不上自己的野心！

Job

关于job hunting，当时有写过相关的blog，个中滋味经历过就深刻了。本科时没有参加什么校招，研二下也没有找相关实习，这算是我涉入职场和社会的第一步吧。归结的话是，校园和社会之间是一条不可见的鸿沟，切莫养尊处优。这也是我来年的愿望清单中重要的一条，我希望自己有更好的offer，让我的技术梦不至于太虚幻。

Technology

这一年，我对未来似乎有越见清晰的规划（不知是不是面试官问出来的orz），结识(单向的/(ㄒoㄒ)/~~)到了一大批业内的大牛:-D滚雪球效应，让我感觉到娱乐是个圈，技术也有个圈哈哈哈，越多关注他们，越是感觉自己的菜鸟本质，也越来越有灯塔标杆效果。因为她们，我感觉自己读书少，感觉自己自学能力弱，感觉自己兴趣爱好少。她们不像是妈妈口中的别人家的孩子，像是…

之所以和job分来，我觉得这和job是不完全并轨的曲线。工作存在一定的不确定性，但理论上技术能力是一条斜率大与1的曲线，但技术也存在个人偏好。最理想状态是工作和技术相辅相成。我也相信兴趣是技术最好的驱动力，我也需要一如既往的多投入时间精力，更有计划性地学习运用。这一年对技术的投入其实不少，项目过程中和没有计划时，因为我感觉自己是兴趣导向型，所以一些事容易拖到deadline。但是一个side project也没有彻底完成，几篇技术blog无法善始善终(但我相信会的)，以上这些警告是在告诉我合理的计划的重要性，不妨试试，或许会感谢做出改变决定的自己。

Academy

这是需要我花时间我却没有足够上心的一块，明年是在校生涯的最后半年，如果不及时规划，你会发现顾此失彼，来不及和学生生活好好告别，来不及边走边体会。

Life & Family

本命年的生活确实过得挺闹腾，这一年每两个月去一次爸妈身边的频率却是许多年来未有的奢侈享受。作为长女，我能感受到自身的压力，不管怎样，这是对自己更好的鞭策。
转眼间，妈妈50岁了，眉眼间好似能感受到她的衰老，今年回去好好过个生日，也想给她的睡眠找找法子。我感觉爸爸有越来越多的不懂的事会来问我，这或许是他越来越依赖我的方式吧。
因为人的不断成长，感觉眼下一切过的特别真切实在。不管是喜欢厌倦，接受排斥都不会后悔。不知是不是本命年的缘故，Follow my heart去了杭州听了张惠妹的演唱会，尽管是一个人/山顶票，我都享受在这段过程，像是给了大学的自己一个交代，也渐渐喜欢上这种游离城市间的看风景/见好友，不断修行的可能。12月10号梁静茹在苏州的演唱会是给自己的treat，这么近地看见我鱼，听她的音，像是早有安排，又像是刻意为之；这一年的两场演唱会，似乎没有克制，但正是这种有触动有爱的感觉让我有了韧性。
想对身边的朋友和老师说声抱歉，没有以最好的姿态相待，要常怀感恩之心，努力改善。

终面结束，知道是和苏小妍说再见的时候/(ㄒoㄒ)/~~，心里很难过，任何的努力在此刻都是苍白的，不能达到该有的技术水平再多的修饰弥补都是徒劳。

  1. 立即执行函数使用块作用域申明来防止污染全局变量

  2. requireJs防止js加载阻塞页面渲染，使用程序调用的方式加载js;其目标是鼓励代码的模块化，使用不同于传统标签的脚本加载步骤，用来加速、优化代码，但主要目的是代码的模块化。
    define用来定义一个模块，require加载依赖模块，并执行加载完后的回调函数；
    加载requireJs脚本的script标签加入data-main属性，这个属性指定的js将在加载require.js后处理我们把require.config的配置加入到data-main后，就可以使每一个页面都使用这个配置，然后页面中就可以直接使用require来加载所有的短模块名
    data-main还有一个重要的功能，当script标签指定data-main属性时，require会默认的将data-main指定的js为根路径，是什么意思呢？如上面的data-main=”js/main”设定后，我们在使用require([‘jquery’])后(不配置jquery的paths)，require会自动加载js/jquery.js这个文件，而不是jquery.js，相当于默认配置了：

    require.config({
    baseUrl: 'js'
})

  • 原始类型有：
    1.number
    整型；浮点型(注意精度问题)
    2.string
    放在引号里的大串字符序列：

    * 单双引号可以交替使用
* 总是记混的几个方法：slice(截取，类似于substring)、split[分割，返回数组]

    3.boolean
    任意的JS值都可以转换为布尔值

    * undefined,null,0,-0,NaN,""(空字符串)--->false
* 除以上外的所有值--->true

    4.null是object类型
    5.undefined未定义

  • 对象类型：Array,Function,Date,RegExp,Error

    对象是属性的集合，每个属性是有‘名/值对’构成；

    普通的Js对象是‘命名值’的无序集合;

    var arr = [1, 2, 3];
arr.length; // 3
arr.length = 6;
arr; // arr变为[1, 2, 3, undefined, undefined, undefined]
arr.length = 2;
arr; // arr变为[1, 2]
var arr = [1, 2, 3];
arr[5] = 'x';
arr; // arr变为[1, 2, 3, undefined, undefined, 'x']
大多数其他编程语言不允许直接改变数组的大小，越界访问索引会报错。然而，JavaScript的Array却不会有任何错误。在编写代码时，不建议直接修改Array的大小，访问索引时要确保索引不会越界.
实际上JavaScript对象的所有属性都是字符串，不过属性对应的值可以是任意数据类型。
JavaScript规定，访问不存在的属性不报错，而是返回undefined.
delete object.key;//删除对象属性
'key' in object;//检测是否拥有某一属性
要判断一个属性是否是object自身拥有的，而不是继承得到的，可以用hasOwnProperty()方法;

  • Tips:

    • 在内存中存储方式不一样：前者存储在栈内存(Stack)中，后者存储在堆内存(heap)中

    • 所以，前者值不可更改，后者可以

      类型转换：

      除了null和undefined不能进行类型转换外，其他所有类型的值都可以转换成其他类型。

    • string/number/boolean直接使用时，存在隐式转换；

    • toString()方法返回一个反映这个对象的字符串；
    • valueOf()方法返回对象本身；
