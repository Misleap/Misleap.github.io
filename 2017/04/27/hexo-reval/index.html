Hello, hexo-reval

Posted on 2017-04-27   |     |   visitors

从接触Hexo开始，就被Hexo的定制化个性主题和插件深深吸引(Good points)。那时就在想，Oneday I must….(then this day comes)~于是，整个人投入多线程工作了，一方面想记录下个人主题Hexo-reval的开发过程；另一方面，因为其中有些技术如模板引擎EJS/Jade/Swig也是没有接触的，过程中就需要边学边实践，好在两者可以互为补充，(so..)针对攻克该Project的key points进行记录和分享。

模板引擎

官方默认的Swig Template engine目前已经停更，这里我选择了EJS。

这里，其实我不禁想到一个问题，在Joylive项目开发的时候，记得当时师兄用的是Freemarker模板引擎，那时是第一次接触Template engine, 觉得用这种代码思想来写前端很了不起，提高前端逼格不说也大大提高了前端的生产力吖！—言归正传，要花点时间了解下Freemarker和本文提到的以上有何差别？区别在于前者用于Java语言环境后者们适用于JS？好像不对…

Stylus

类似于SASS、LESS，Stylus以接近脚本的方式去写CSS代码。CSS的弱点是静态化，以上预处理框架旨在创建健壮和动态的CSS，真正提高前端CSS的开发效率，算是前端开发人员的福音~

以上是Hexo主题制作主要涉及的两个技术。另外，我们知道Hexo主题一般包括几个页面：

页面 模板
首页 index.ejs
归档页 archive.ejs
文章详情页 post.ejs
页面详情页 page.ejs
标签页 tag.ejs
分类页 category.ejs

Hexo官网对所有变量(eg:global/site/page etc.)进行了介绍，这对模板的理解和书写有很好的参考意义。个人认为只要若干重要的变量理解了就基本OK, 这样就知道页面各部分展示内容。
